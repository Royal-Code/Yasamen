# Instrução Operacional - Refinar uma Spec

> Use este arquivo como base em novos chats para que a IA fortaleça uma spec existente antes da implementação.

## Fontes Obrigatórias

Antes de refinar a spec, a IA deve ler:

- `.ai/templates/lib/spec/requirements.md`
- `.ai/templates/lib/spec/design.md`
- `.ai/templates/lib/spec/tasks.md`
- `.ai/templates/lib/spec/delivery.md`
- `.ai/ui-map/ui-map.md`
- `.ai/roadmap/ui-plan.md`
- `.ai/roadmap/components-plan-list.md`
- `.ai/guides/rules/cross-cutting-component-decisions.md`
- os guides listados em `Guides aplicados`

## Regras

- Tornar a spec mais consistente, específica, executável e rastreável.
- Garantir que `cross-cutting-component-decisions` esteja em `Guides aplicados` quando a spec for de componente.
- Alinhar a spec com os guides aplicáveis, incluindo:
  - `css-visual-contract.md`;
  - `outlet-patterns.md`;
  - `navigation-patterns.md`;
  - `component-composition-and-dependencies.md`;
  - `form-components-lightweight.md`, quando for formulário ainda aberto.
- Validar explicitamente todos os checkpoints do guide `cross-cutting-component-decisions.md`.
- Quando houver pacote novo, registrar explicitamente a dependência de `.ai/instructions/expand/create-library-project.md`.
- Preservar o que já estiver bom.
- Usar sempre acentuação.
